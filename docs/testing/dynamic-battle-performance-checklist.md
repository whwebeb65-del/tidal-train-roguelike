## 稳定性检查

- [ ] 自动画质只降低视觉预算，没有改变敌人、炮弹判定、伤害、掉落或奖励。
- [ ] 高 / 中 / 低画质在同种子、同输入下产生相同战斗结局。
- [ ] 页面隐藏后帧循环停止；恢复后不补算离屏时间。
- [ ] 恢复页面时先恢复音频，再由玩家主动继续战斗。
- [ ] 主炮、命中、击杀、技能、Boss 和结算声音无爆音、重叠或持续残留。
- [ ] 旋转屏幕或改变视口后 Canvas 清晰，HUD 无横向滚动。
- [ ] 第一局回站后活动帧循环为 0，战斗实体和效果计数归零。
- [ ] 第二局回站后监听器回到首局前基线，活动音频调度器不超过 1。
- [ ] 两局后内存回落到稳定区间，没有逐局持续增长。
- [ ] 控制台和诊断快照无未捕获异常。

## 判定与限制

- 桌面 Chrome 数据用于发现回归，不能写成“抖音真机稳定 60 FPS”的结论。
- 发布前至少补测一台目标低端安卓机和一台主流安卓机，并在抖音开发者工具中记录同一套指标。
- 若平均表现正常但存在可感长帧，应记录出现阶段和复现条件，不能只填平均值掩盖卡顿。
- 若两局后资源没有回落，应先修复生命周期或对象释放问题，再扩大内测。
