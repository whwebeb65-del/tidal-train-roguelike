# 《最后一班：潮汐列车》原型试玩脚本

## 目的

验证玩家能否在短时间内理解“预约领奖—车站升级—每日试炼—组队—出发—战斗—选择—互动—军团贡献—首通”的核心闭环，并确认固定种子、内测资格、礼包码、重复点击奖励、三货币、地图开放和异步协作不会造成明显困惑。

## 准备

1. 在项目根目录运行 `npm run dev`。
2. 打开 `http://127.0.0.1:4173/`。
3. 点击底部“清空本地存档”，确保从 1 级车站、漂流郊区开始。
4. 如需移动端观察，将浏览器窗口缩窄到约 390px 宽。

## 主流程

1. 记录从页面打开到第一次有效点击的时间；目标是 30 秒内点击“驶向漂流郊区”。
2. 进入战斗后，观察玩家是否能找到三条航道、自动开炮、汽笛共鸣、维修车厢和潮汐爆发按钮；先连续开炮，确认动能和连击会上升。
3. 在“同局互动奖励”中连续点击“打捞废弃信标”两次，确认齿轮从 0 变为 16，按钮显示 `2/2` 且禁用。
4. 点击“给漂流者递水”和“点亮潮汐信号”，确认航线徽记和星票分别增加，且每项只能领取一次。
5. 先用“维修车厢”恢复一次生命，再用技能或继续开炮击败当前敌人；确认维修按钮本节点只可使用一次，选择一个乘客或模块后进入路线选择。
6. 依次选择路线、奖励和下一条路线，观察风险、节点类型和构筑提示是否足够清楚。
7. 进入 Boss，观察节点修饰标签；连续攻击积满 100 动能后点击“潮汐爆发”，确认 Boss 击破并首通结算显示：400 齿轮、10 航线徽记、3 星票。
8. 回到车站，连续升级两次，确认车站达到 3 级且“旧港口”从锁定变为开放。
9. 点击“模拟购买 60 星票”，确认商店边界只在 Mock 中生效，并观察星票变化。

## 重点观察

- 第一次有效点击是否明显；
- 互动按钮的“本局次数”和“已领完”是否容易理解；
- 三种货币是否能区分用途；
- 首通奖励与重复通关奖励是否清楚；
- 地图锁定条件（车站等级）是否可预测；
- 移动端窄屏下按钮是否仍可点击；
- 玩家是否愿意在首通后再次挑战不同构筑。
- 节点修饰是否能让玩家理解不同的攻击、技能与维修选择；新节点是否重置维修、爆发、连击和动能。
- 窄屏设备上战斗状态栏和四个动作按钮是否仍然清晰、可点击。

## 通过标准

- 试玩者无需口头解释即可完成第一次出发、一次互动、一次奖励选择；
- 重复点击不会重复发放，达到上限后有明确反馈；
- 首通奖励在结算页和资源栏中一致；
- 车站升级与地图开放状态刷新正确；
- 浏览器控制台无阻断性错误。
- 同一单局只能贡献一次军团进度，同一里程碑只能领取一次，分享招募不会直接增加钱包货币。
- 普通战斗不应产生持续计时器或持续 `requestAnimationFrame` 业务循环；每次动作只触发一次状态更新和一次 UI 重绘。
- 性能观察以“无长任务、无明显掉帧、动画可在窄屏关闭”为目标；本脚本不替代抖音目标机型的真机 60 FPS 验证。

## 异步组队与军团远征回归路径

1. 清空本地存档，在车站找到“联合作战中心”，点击加入“潮汐灯塔团”。
2. 依次选择领航员和炮术员，确认队伍显示 `2/2`，摘要显示“开场动能 +20 · 行动伤害 +5”。
3. 尝试选择工程师，确认出现“列车队只有两个支援位”的反馈，原有两名队友不被替换。
4. 开始一局，确认战斗初始动能为 `20/100`，普通攻击显示的伤害比无炮术员时增加 5，支援栏显示两名队友。
5. 完成一局胜利，确认结算页显示本局远征贡献，回到车站后周进度同步增加。
6. 达到 20 贡献后领取“前哨补给”，确认只增加 30 齿轮；按钮变为“已领取”，快速重复点击不再发奖。
7. 点击“分享列车队招募卡”，确认完成反馈明确说明分享不直接发放货币，现有军团贡献和队伍选择不改变。
8. 将浏览器缩窄到约 390px，确认三个里程碑、三张支援卡和分享按钮改为单列且都可点击。

正式服务端回归还必须验证：不同玩家加入、邀请归因、周周期切换、并发贡献、重复请求、成员退出、敏感词、举报和奖励补发；本地 Mock 不能替代这些测试。

## 内测资格、开服礼与礼包码回归路径

1. 清空本地存档，在车站找到“首班预约中心”，确认显示 `FOUNDERS / 3,000` 和“原型活动开放中”，且没有伪造实时剩余名额。
2. 点击“申请内测资格”，确认状态变为“内测资格已锁定”，三种货币仍为 0。
3. 点击“领取先行者补给”，确认资源变为 60 齿轮、2 航线徽记、1 星票，并出现“潮汐先行者”徽章。
4. 点击“领取开服列车长礼”，确认资源累计为 248 齿轮、8 航线徽记、4 星票，并出现“开服列车长”徽章。
5. 在礼包码输入框输入带空格和小写的 ` tide2026 `，按回车或点击兑换；确认资源累计为 314 齿轮、10 航线徽记、4 星票。
6. 再次尝试申请资格、领取两类礼包和兑换 `TIDE2026`，确认按钮保持已领取或出现重复提示，资源不再增加。
7. 刷新页面，确认内测资格、两个已领取按钮、两个徽章和礼包码核销状态全部保留。
8. 输入空字符串和未知码，确认分别提示“请输入礼包码”和“礼包码无效”，反馈中不回显原始未知输入。
9. 将浏览器缩窄到约 390px，确认两张活动卡、礼包码输入框和兑换按钮改为单列，页面无横向滚动。

正式服务端还必须验证：3,000 名额并发抢占、候补与释放、服务器时间、礼包码总库存、渠道批次、账号封禁、重复请求幂等、资产事务、运营撤回和客服补发。本地活动状态只能验证单账号体验。

## 每日潮汐试炼回归路径

1. 清空本地存档，确认“今日潮汐试炼”可预览，但按钮显示“车站 Lv.2 开放”。记录日期、固定种子与今日规则。
2. 只领取“开服列车长礼”，确认钱包为 188 齿轮、6 航线徽记、3 星票；花费 80 齿轮升级到车站 Lv.2，确认钱包为 108/6/3，试炼按钮变为“开始今日试炼”。
3. 刷新页面，确认日期、种子和今日规则不变化，车站等级仍为 2。
4. 开始今日试炼，确认战斗页显示 `DAILY TRIAL`、同一日期、种子和规则；对照规则卡验证敌人生命、列车最大生命、开场动能或行动伤害至少一项真实变化。
5. 确认普通“同局互动奖励”按钮没有出现，替换为“每日试炼关闭常规互动货币”的公平性说明。
6. 不使用复活，连续点击“模拟受击”至失败并放弃；确认结算显示本局 20 分、今日最佳 20、尝试 1、无救援成绩。
7. 回到车站，确认钱包仍为 108/6/3，军团贡献没有变化，普通地图首通没有记录；“今日出发”里程碑按钮变为“领取”。
8. 领取“今日出发”，确认钱包只变为 138 齿轮、6 航线徽记、3 星票；按钮变为“已领取”，重复点击不再增加。
9. 再开始一局试炼，使用广告或分享复活后结算，确认显示“救援成绩 · -25”，且最低提交分不低于 20。
10. 在试炼结算点击“分享同种子试炼”，确认完成提示说明分享不直接发货币，钱包和最佳分不改变。
11. 将浏览器缩窄到约 390px，确认试炼入口、规则、两档里程碑、战斗提示和结算按钮都为单列或自然换行，页面无横向滚动。

正式服务端回归还必须验证上海时区跨日、并发成绩提交、篡改种子、伪造战斗、复活归因、榜单分层、封禁账号和资产事务。本地日期与分数只能验证体验流程。

## 复活与技能刷新回归路径

1. 清空本地存档，开始一局普通战斗，连续点击“模拟受击 · -35”至失败。
2. 点击“看广告复活”，确认生命恢复到 60/100，并回到当前战斗。
3. 再次受击至失败，点击“分享战绩复活”，确认生命恢复到 50/100，并回到当前战斗。
4. 再次受击至失败，确认广告复活和分享复活都显示“已用”，点击放弃并确认进入 `RUN SETTLED` 结算页。
5. 开始新局，先释放一次技能，确认技能充能变为 0/1 且出现“看广告刷新技能”。
6. 点击“看广告刷新技能”，确认技能充能恢复为 1/1；再次检查本局刷新入口不可重复使用。

## 观察记录

- 取消广告或分享、平台回调失败时，不应消耗对应次数或技能刷新次数；
- 复活成功后应保留当前敌人生命、路线和同局互动奖励；
- 分享卡应包含地图、深度、乘客、模块、失败原因和明确 CTA；
- 快速重复点击复活或刷新按钮不得重复发奖。
