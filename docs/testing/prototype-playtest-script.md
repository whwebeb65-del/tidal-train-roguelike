# 《最后一班：潮汐列车》动态原型试玩脚本

## 目的

验证玩家能否在没有口头教学的情况下理解“选择列车长—整备—自动战斗—主动技能—肉鸽强化—互动奖励—精英/Boss—结算—长期养成”的核心闭环，并观察画面、声音、节奏和付费展示是否清楚但不过载。

本脚本测试的是 Web 原型。广告、分享、组队和支付仍是本地 Mock，不代表抖音、应用商店或真实支付渠道已经接通。

## 准备

1. 在项目根目录运行 `npm run dev`，打开命令行显示的本地地址。
2. 点击“重置”清空本地存档，从 Lv.1 车站和“漂流郊区”开始。
3. 记录设备、系统、浏览器、视口、设备像素比、音乐/音效状态和画质设置。
4. 首轮不要解释界面；试玩者卡住 30 秒后才允许给一句提示，并记录提示位置。
5. 自动化基线可运行：`npm run build` 后执行 `npm run smoke:browser`。

## 首次会话主流程

1. 让玩家自行选择女或男列车长，记录完成选择所需时间和选择原因。
2. 让玩家依次进入车站、角色、装备、军团、商店，确认五个场景是真实切换，而不是同一页面只换标题。
3. 回到车站点击出发。记录玩家首次注意到“主炮会自动开火”的时间；不要提示玩家点击画面开炮。
4. 观察玩家能否分辨普通怪、护甲怪、精英和 Boss，以及炮弹、命中、击杀、掉落、受击和角色动作。
5. 让玩家使用“潮汐齐射”“泡泡屏障”“极潮爆发”，分别记录使用次数、理解用途所需时间和误触情况。
6. 在约 18 秒出现“打捞漂流信标”后连续领取两次，确认齿轮共增加 16；第三次不能继续发奖，并有明确反馈。
7. 每次出现三选一时让玩家自行选择。记录三次强化选择、选择理由，以及强化后是否能感受到构筑变化。
8. 观察普通怪、精英、Boss 三段节奏。目标单局约 3–5 分钟，不能长时间没有新事件，也不能持续高密度到无法辨认。
9. 若首次失败，让玩家选择“看广告复活”或放弃结算；确认 Mock 广告完成后只复活一次，并保留当前战斗进度。
10. 通关后核对首通高额奖励：400 齿轮、10 航线徽记、3 星票；结算层与顶部资源栏必须一致。
11. 回到车站后立即询问是否愿意再开一局，不主动引导。若继续，记录第二局使用的不同构筑和重复通关奖励。

## 动态演出观察

- 自动主炮是否无需教学即可理解。
- 三个主动技能的图标、名称、冷却和演出是否容易区分。
- 三次肉鸽选择是否清楚，倒计时和广告刷新是否会制造误操作。
- 击杀特效、伤害数字、掉落、屏障、精英入场和 Boss 转场是否“精致但不花哨”。
- 背景、角色、敌人和 HUD 是否有明确层级；窄屏下重要按钮不能被遮挡。
- 音乐、主炮、命中、击杀、技能、受击、胜负音效是否匹配画面且不会爆音。
- 暂停、切到后台再回来后，战斗必须停住并要求玩家主动继续，不能补算离屏时间。
- 自动画质降低时，只能减少粒子、残影、闪光、伤害数字和 DPR，不能改变敌人、炮弹判定、伤害或奖励。

## 长期玩法与留存回归

1. 完成每日签到，确认同一天不能重复领取，跨日提示清楚。
2. 领取内测资格、先行者补给和开服礼，刷新页面后确认状态保留。
3. 输入 ` tide2026 `，确认礼包码会规范化并只核销一次；未知码的错误提示不得回显原始输入。
4. 升级车站，确认地图按等级逐步开放，未达条件时能看懂解锁要求。
5. 加入军团、选择两名支援成员、完成一局并领取里程碑；重复点击不得重复发奖。
6. 打开每日潮汐试炼，确认日期、固定种子和规则稳定；试炼结算不得误发普通首通奖励。
7. 查看皮肤、装备和礼包。确认属性叠加展示清楚，并明确这是本地 Mock 商店，不得出现“真实付款已完成”的误导。
8. 检查广告复活、广告刷新技能、广告刷新三选一、重复通关追加奖励和分享入口；取消或失败不能消耗次数。

## 通过标准

- 80% 以上试玩者可在 30 秒内完成列车长选择并找到出发入口。
- 试玩者无需提示即可意识到自动开炮，并至少主动使用两个技能。
- 三次升级选择、普通怪/精英/Boss 节奏和结算闭环可完整完成。
- 同局重复互动只发两次奖励，首通和重复通关结算均无重复发放。
- 五个场景独立切换，360–430px 宽视口无横向滚动和不可点击按钮。
- 连续两局后无未捕获异常、重复声音、残留战斗动画或明显越来越卡的问题。
- 玩家能区分三种货币、首通奖励、重复奖励、永久属性与局内强化。
- 至少 60% 完成首局的试玩者表示愿意开始第二局；该指标用于迭代，不作为虚构的商业承诺。

## 必须另行验证的边界

- 桌面 Chrome 的帧率与自动化 smoke 不能替代抖音开发者工具和真实安卓目标机测试。
- 真实广告回调、分享归因、支付验单、退款、礼包库存、军团并发和账号风控必须接入正式服务端后再测。
- 不得用个人收款码替代平台支付结算；正式收益账户需在对应平台商户后台配置。
