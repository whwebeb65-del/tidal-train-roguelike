# 《最后一班：潮汐列车》现代精品Q版与列车长成长系统设计

- 日期：2026-07-16
- 项目：project-002-tidal-train-roguelike
- 状态：已完成方案确认，待用户书面审阅
- 适用范围：Web 可玩原型、后续抖音小游戏客户端

## 1. 设计结论

本轮采用“克制版现代精品Q风”，替换当前深色复古水城与重古铜界面。核心视觉由约 3.5 头身的男女列车长、珍珠白泡泡列车、悬浮珊瑚岛、液态轨道和可爱但具有攻击性的潮兽组成。

首次进入游戏时，玩家从男、女列车长中选择一名作为当前角色。两者基础能力完全一致，可在衣柜中随时切换。皮肤和装备都是永久成长来源：

- 皮肤：账号拥有后立即提供收藏属性，不要求装备，所有唯一皮肤的属性永久相加。
- 装备：只有当前穿戴的装备生效，可升级、升星、洗练词条并触发套装效果。
- 男女列车长共享账号进度、皮肤收藏加成和装备战力。
- 皮肤与装备属性在普通推图、每日试炼、排行榜、军团活动和后续新增玩法中全部生效，不设置公平模式或属性归一。

本规范明确替代旧设计中的以下约定：

- `2026-07-15-tidal-train-design.md` 中“原型仅一名列车长”和“永久成长不形成付费战力”的相关约定。
- `2026-07-16-revenue-foundation-v1-design.md` 中“付费外观不增加战力”的相关约定。
- `docs/product/player-feature-roadmap.md` 中“付费内容仅提供非战力奖励”的相关约定。

其他关于确定性商品、交易幂等、服务端验单、无付费随机宝箱和激励广告自愿触发的约定继续有效。

## 2. 本轮目标与非目标

### 2.1 目标

1. 把现有网页原型改造成风格统一、人物驱动、适合竖屏传播的精品Q版界面。
2. 提供男、女两名基础能力一致的列车长，并支持首次选择和后续切换。
3. 建立可持续售卖的皮肤目录、衣柜、永久收藏属性和累计战力展示。
4. 建立四槽装备系统，让装备承担主要战力和流派差异。
5. 保留现有战斗、地图、签到、每日试炼、军团、礼包码、广告和购买流程，不重写已经验证的游戏内核。
6. 保持事件驱动渲染和移动端性能降级，目标为主流设备 60 FPS、低性能设备不低于 45 FPS。

### 2.2 非目标

- 本轮不接入真实抖音、微信、Apple 或 Google 支付。
- 本轮不搭建生产服务器、真实排行榜和军团服务端。
- 本轮不加入付费随机抽取、概率宝箱或现金奖励。
- 本轮不制作完整骨骼动画、剧情演出和每张地图的全套动态场景。
- 本轮不新增大批关卡、伙伴或 Boss，先完成一个可验证的精品竖切版本。
- 当前收款码、个人支付信息和支付密钥不得进入客户端、仓库或埋点。

## 3. 视觉系统

### 3.1 风格基准

- 角色比例：约 3.5 头身，大眼、清晰轮廓、年轻成人气质，不做幼儿化表达。
- 主色：午夜靛蓝与珍珠白。
- 辅色：低饱和海沫绿。
- 强调色：珊瑚橙只用于围巾、攻击、重要通知和关键按钮。
- 禁用方向：旧欧洲水城、厚重蒸汽朋克、巨型齿轮、满屏古铜花纹、高饱和彩虹配色和无意义泛光。
- 材质：哑光织物、珍珠列车外壳、柔和珐琅按钮、克制的透明水体。

### 3.2 世界与角色识别

女列车长保留午夜蓝短发、海绿色眼睛、珊瑚围巾和潮灯。男列车长使用深靛蓝短发、少量海沫色发束、珊瑚领巾和潮汐罗盘护腕。两者使用相同的世界制服体系，但姿态、发型和手持装置必须能形成独立剪影。

海獭机械师负责维修与炮台表现，水母医师负责治疗与护盾表现。首轮敌人包括三种普通潮兽和一个 Boss，造型遵循“大轮廓、少零件、可爱但有攻击性”的原则。

泡泡列车采用珍珠白胶囊车身、透明泡泡窗和魔鬼鱼鳍形车顶。地图背景使用大面积天空、水面和少量悬浮岛剪影，减少碎珊瑚、泡泡和装饰物。

### 3.3 界面层级

1. 车站主页：列车长是视觉中心，列车与伙伴构成场景；只保留一个主要出发按钮、一个当前任务提示、一个礼物提醒和五项底部导航。
2. 战斗页：上方只显示路线进度和敌人生命；中部保留清晰战场；底部固定三张核心技能卡和伙伴状态。
3. 路线、奖励与结算：继续使用现有流程，但改为大图标、低装饰、统一圆角和统一按压反馈。
4. 签到、军团、礼包码、商店：功能保持不变，统一改为蓝白软珐琅卡片，不与主场景争夺视觉焦点。
5. 所有可点击区域最小尺寸为 44 × 44 CSS 像素，360–430 像素宽度下不得横向溢出。

## 4. 列车长系统

### 4.1 基础角色

首轮提供：

- `captain-tide-female`
- `captain-tide-male`

两名角色共享相同的基础生命、伤害、动能、技能和装备槽。角色选择只改变立绘、头像、待机动作、技能表现和可装备外观，不改变基础数值。

首次进入且 `selectedCaptainId` 为空时显示角色选择页。玩家确认后进入车站；之后可从衣柜随时切换，不消耗货币，也不设冷却。

### 4.2 皮肤外观

皮肤定义可以包含男款、女款或两者。购买页必须明确显示包含哪些款式。即使某套皮肤只有一名列车长可以穿戴，其收藏属性仍然作用于整个账号和当前选择的任意列车长。

装备皮肤只影响当前显示外观；收藏属性只依据唯一拥有的皮肤 ID 计算，与当前穿戴状态无关。

## 5. 皮肤永久叠加规则

### 5.1 所有权与叠加

- 每个唯一 `skinId` 最多计入一次收藏属性。
- 皮肤拥有后立即生效，不要求穿戴。
- 所有已拥有皮肤按属性类别直接相加，不设总上限。
- 男女列车长共享同一组 `ownedSkinIds` 和累计属性。
- 所有模式使用同一套皮肤属性，不建立公平模式。
- 重复交易、重复回调或重复领取不得重复增加属性。

### 5.2 首轮稀有度基准

| 稀有度 | 单套固定收藏属性 | 总增幅参考 |
|---|---|---:|
| 基础 | 无属性 | 0% |
| 普通 | 列车最大生命 +0.5% | 0.5% |
| 稀有 | 列车最大生命 +0.5%，行动伤害 +0.5% | 1.0% |
| 史诗 | 列车最大生命 +0.75%，行动伤害 +0.5%，齿轮获取 +0.25% | 1.5% |
| 典藏 | 列车最大生命 +1.0%，行动伤害 +0.5%，齿轮获取 +0.5% | 2.0% |

皮肤的实际词条写入目录配置，不由界面根据稀有度临时推算。未来可以增加其他固定词条，但一套皮肤不得同时包含随机数值。

### 5.3 展示规则

衣柜和商店必须同时显示：

- 皮肤完整外观和适用列车长。
- 稀有度与固定属性。
- 已拥有、未拥有和当前穿戴状态。
- 购买后账号累计属性的变化。
- 当前全部皮肤提供的总收藏属性。

不得使用“可能获得”“最高可得”等模糊文案。皮肤购买是确定性商品。

### 5.4 首轮皮肤目录

首轮至少实现三套皮肤系列，每套均包含男、女款式：

| 皮肤 ID | 名称 | 获取方式 | 稀有度 |
|---|---|---|---|
| `skin-tide-base` | 潮汐制服 | 默认拥有 | 基础 |
| `skin-seafoam-departure` | 海沫启航 | 七日成长活动固定领取 | 普通 |
| `skin-aurora-whale-song` | 极光鲸歌 | 确定性付费商品，Web Mock 展示价 ¥30 | 典藏 |

`skin-aurora-whale-song` 一次购买同时解锁男、女款式，只计算一次典藏收藏属性。正式价格由目标平台商品后台配置，客户端只展示平台下发或版本配置的价格文本。

## 6. 装备成长系统

### 6.1 四个装备槽

| 槽位 | 主要作用 |
|---|---|
| 主炮 | 行动伤害、技能伤害、暴击类词条 |
| 车体 | 最大生命、护盾、受伤减免 |
| 动力核 | 开场动能、动能获取、技能循环 |
| 潮汐仪 | 齿轮收益、维修、特殊流派词条 |

只有当前穿戴的四件装备生效。装备跟随账号，不绑定男、女列车长；切换角色后继续使用同一套装备。

### 6.2 装备结构

每个装备实例包含：

- `instanceId`
- `definitionId`
- `slot`
- `rarity`
- `level`，首轮范围 1–20
- `stars`，首轮范围 0–5
- 固定主属性
- 最多两个可洗练副词条
- 可选 `setId`

升级消耗齿轮并提升主属性；升星消耗对应碎片并提高基础成长系数；洗练只改变副词条，不改变槽位、稀有度和套装。两件和四件同套装备分别触发套装效果。

装备可以通过玩法掉落、活动奖励和确定性商品包获得。当前范围不加入付费随机装备箱。

### 6.3 首轮装备目录

首轮实现两套、共八件装备，每个槽位各两件：

- 潮泡守望套：潮泡主炮、珍珠车体、静流动力核、海沫潮汐仪。
- 珊瑚突击套：珊瑚主炮、轻浪车体、急流动力核、追潮仪。

套装效果固定为：

- 潮泡守望 2 件：最大生命 +3%。
- 潮泡守望 4 件：维修恢复量 +6。
- 珊瑚突击 2 件：行动伤害 +3%。
- 珊瑚突击 4 件：开场动能 +15。

默认存档获得 1 级 0 星的完整潮泡守望套。珊瑚突击套通过普通推图和确定性商店逐步获得，用于验证换装和流派差异。

## 7. 战力计算与模式接入

### 7.1 聚合顺序

新增统一的 `ProgressionStatService`，负责聚合：

1. 基础战斗数值。
2. 当前穿戴装备的固定值。
3. 当前穿戴装备的百分比。
4. 所有已拥有皮肤的百分比收藏属性。
5. 当前玩法的临时规则和局内构筑。

同类百分比先相加，再统一乘算；最终显示值向下取整：

```text
最终最大生命 =
floor((基础生命 + 装备固定生命) ×
      (1 + 装备生命百分比 + 皮肤生命百分比))

单次行动伤害 =
floor((行动基础伤害 + 装备固定伤害 + 玩法固定加成) ×
      (1 + 装备伤害百分比 + 皮肤伤害百分比))

齿轮结算 =
floor(基础齿轮 ×
      (1 + 装备齿轮百分比 + 皮肤齿轮百分比))
```

如果玩家没有额外皮肤和装备，计算结果必须与当前原型完全一致。

### 7.2 全模式生效

普通推图、首次通关、每日固定种子试炼、排行榜、军团 Boss、军团活动和后续新增玩法全部调用同一个属性聚合服务。固定种子只保证路线和规则一致，不保证不同账号战力一致。

未来新增模式时，除非用户重新修改产品决策，否则不得单独关闭皮肤或装备属性。

## 8. 技术架构

### 8.1 领域模块

新增以下独立模块：

- `src/domain/captain/CaptainCatalog.ts`：列车长静态定义。
- `src/domain/captain/CaptainProfileSystem.ts`：首次选择、切换和皮肤穿戴。
- `src/domain/skin/SkinCatalog.ts`：皮肤定义、款式、价格展示和固定属性。
- `src/domain/skin/SkinCollectionSystem.ts`：唯一所有权去重和永久属性聚合。
- `src/domain/equipment/EquipmentCatalog.ts`：装备与套装静态定义。
- `src/domain/equipment/EquipmentSystem.ts`：穿戴、升级、升星、洗练和套装聚合。
- `src/domain/progression/ProgressionStatService.ts`：把皮肤、装备和玩法数值合并为战斗输入。

界面层只提交命令并展示结果，不直接修改战力、资产或所有权。

### 8.2 存档升级

主存档从版本 2 升级为版本 3，新增：

```ts
selectedCaptainId: 'captain-tide-female' | 'captain-tide-male' | null
ownedSkinIds: string[]
equippedSkinIds: Record<string, string>
equipmentInventory: EquipmentInstance[]
equippedEquipmentIds: Record<EquipmentSlot, string | null>
equipmentFragments: Record<string, number>
```

版本 2 迁移到版本 3 时：

- 恢复两名列车长的基础皮肤。
- `selectedCaptainId` 保持 `null`，首次打开新版时要求选择。
- 旧 `ownedCosmeticIds` 继续保留；已知可迁移外观映射到新皮肤，未知 ID 不丢失。
- 添加一套 0 星 1 级的新手装备，保证现有玩家可以立即继续游戏。
- 初始化空的装备碎片库存；后续玩法和确定性商品按装备定义 ID 发放碎片。
- 数组去重，非法目录 ID 忽略并记录非敏感诊断事件。

### 8.3 商店与购买

`ProductReward` 增加 `skinIds` 和 `equipmentDefinitionIds`。只有平台返回已验证交易后才发放所有权。购买服务继续使用 `processedTransactionIds` 保证幂等。

原有“深海引擎涂装不增加战力”商品保持旧行为；新列车长皮肤使用新皮肤目录和收藏属性。真实支付仍然需要目标平台 SDK、可信服务端验单、订单恢复、退款与补单。

### 8.4 异常处理

- 未知列车长：回到首次选择页。
- 已穿戴皮肤缺失：回退对应基础皮肤。
- 未知皮肤 ID：不计算属性，但保留原始所有权记录供后续恢复。
- 非法装备实例：不穿戴、不计属性，并从可操作列表隔离。
- 重复皮肤发放：返回幂等成功，不重复加成。
- 购买取消或失败：不修改资产，按钮恢复可操作。
- 图片加载失败：显示对应角色或物品的统一剪影占位，不阻断流程。

## 10. 资源与性能预算

### 10.1 资源拆分

- 场景背景、列车、列车长、伙伴、敌人、特效和 UI 分层导出。
- 无透明通道的大图优先使用 WebP；角色和特效透明素材使用压缩 PNG 或受支持的透明 WebP。
- 首屏只加载当前列车长、基础伙伴、车站背景和核心 UI。
- 另一性别列车长、衣柜皮肤、装备详情和非当前地图按入口延迟加载。
- 概念稿只用于美术方向，不直接作为最终运行时整屏贴图。

### 10.2 预算

- 首屏关键压缩资源目标不超过 1.5 MB。
- 单张场景背景目标不超过 350 KB。
- 单名列车长首屏分层资源目标不超过 450 KB。
- 单套 UI 图集目标不超过 512 KB。
- 动画优先使用 `transform` 和 `opacity`，不建立持续业务 `requestAnimationFrame`。
- 低性能设备关闭景深、非关键粒子、背景漂浮和多层水光。
- 支持 `prefers-reduced-motion`。

60 FPS 与低端机 45 FPS 是目标，不是当前网页构建即可证明的结果；正式发布前必须在抖音开发者工具和至少两档安卓真机上验证。

## 12. 测试与验收

### 12.1 单元测试

- 男、女列车长基础能力一致。
- 首次选择、免费切换和皮肤穿戴规则正确。
- 相同皮肤 ID 只计算一次，不同皮肤全部相加。
- 皮肤属性不因卸下或切换列车长失效。
- 所有模式获得相同的皮肤与装备属性输入。
- 四槽装备穿戴限制、升级、升星、洗练和套装规则正确。
- 属性计算顺序和取整符合公式。
- 版本 2 到版本 3 无损迁移。
- 已验证交易、重复交易和重复商品发放保持幂等。

### 12.2 Web 回归

- 360、390、412 和 430 像素宽度下无横向溢出。
- 首次选择后进入车站，刷新后保持选择。
- 男女切换、皮肤穿戴和装备更换立即更新主页、战斗头像和属性。
- 商店清晰展示固定商品、固定属性、价格和拥有状态。
- 图片失败时占位图不破坏布局。
- 键盘焦点、触摸目标、降低动态效果和颜色对比可用。
- 现有签到、每日试炼、军团、礼包码、广告和购买流程继续工作。

### 12.3 自动验证

- `npm test`
