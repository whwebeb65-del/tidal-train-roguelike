# 《最后一班：潮汐列车》手绘生活世界精品竖切设计

- 日期：2026-07-18
- 项目：project-002-tidal-train-roguelike
- 状态：设计已逐段确认，等待书面规范审阅
- 首轮范围：车站主页、出发演出、第一场战斗

## 1. 设计结论

本轮采用“手绘赛璐璐生活绘本”方向，替换当前偏通用 AI 概念图的光滑梦幻风格。保留男女列车长、泡泡列车、海獭机械师、水母站务员及潮汐世界的身份设定，但重新统一背景、列车、人物、伙伴、特效和公共 UI 的造型语言。

主场景设定为暖色傍晚的“潮汐末班站”。生命力来自少量具有用途的生活事件，而不是增加无意义泛光、漂浮装饰或持续满屏粒子。首轮先完成“车站主页 → 出发演出 → 第一场战斗”的精品竖切，验证风格、动态、性能与玩家反馈后，再扩展到签到、商店、衣柜、军团等页面。

## 2. 问题诊断

当前版本的主要视觉问题是：

1. 背景由大面积蓝色天空、水面和悬浮岛构成，空间干净却无人生活，缺少地点记忆与日常叙事。
2. 背景、列车和人物像三张分别完成的宣传图叠放，光源、接触阴影、比例和材质联系不足。
3. 人物头发、列车外壳和界面面板均使用光滑高光与渐变，材质趋同，形成塑料感。
4. 现有漂浮、呼吸和缩放动画多为固定循环，运动存在但没有事件因果，容易显得机械。
5. 透明玻璃面板、发光描边与通用圆角按钮削弱世界特色，界面没有成为车站的一部分。

本轮不得通过堆叠更多水晶、泡泡、霓虹、悬浮岛或装饰粒子解决上述问题。

## 3. 目标与非目标

### 3.1 目标

- 让玩家在一秒内认出车站、列车长、列车和出发入口。
- 让角色、列车、背景和 UI 看起来属于同一部二维动画。
- 通过有用途、不同步、不过密的生活事件提升生动感。
- 让出发演出自然连接战斗，并强化列车持续前进的感受。
- 保留现有存档、成长、商业化、签到、军团和战斗数值逻辑。
- 保持移动端清晰度、资源预算、性能降级和无障碍支持。

### 3.2 非目标

- 首轮不重绘签到、商店、衣柜、装备、军团和活动页面的完整场景美术。
- 首轮不新增关卡、敌人、数值、付费商品或存档字段。
- 不制作完整骨骼动画、长剧情过场、角色配音或大规模逐帧动画。
- 不重写已经验证的战斗内核、列车运动控制器和音频生命周期。
- 不把概念图整屏直接作为不可拆分的运行时贴图。

## 4. 视觉语言

### 4.1 线条与色块

- 轮廓使用偏深蓝而非纯黑，外轮廓略粗，内部结构线较细。
- 线条允许轻微粗细变化和不完全对称，避免矢量模板般绝对平滑。
- 明暗以两到三档清晰色块表达，减少连续渐变、镜面反射和密集高光。
- 纹理只使用轻量纸张颗粒、干刷边缘和局部补漆痕迹，不覆盖文字与战斗判读区域。

### 4.2 色彩与光线

- 主场景时间为暖色傍晚：珊瑚橙夕阳、暖白站灯、深蓝阴影和低饱和海绿水面。
- 暖光集中在人物脸部、列车门窗、站牌和主要操作入口；背景远处降低对比。
- 角色、列车和地面共享同一夕阳方向与接触阴影，不允许独立棚拍式打光。
- 战斗保持同一时间段，但提升前景轮廓与攻击色块对比，保证判读。

### 4.3 人物与伙伴

- 男女列车长保留深蓝头发、珊瑚围巾、潮灯、罗盘和制服身份特征。
- 比例保持约 3.5 头身，使用大轮廓、少零件和略不对称站姿。
- 头发改用成组色块，衣服加入少量褶皱、缝线与使用痕迹，取消过密发丝和首饰堆叠。
- 海獭机械师突出工具包、检修手套和油渍；水母站务员突出报站牌、灯帽和柔软形变。
- 角色脚底必须有接触阴影；角色色温随场景改变，避免贴纸感。

### 4.4 泡泡列车

- 保留圆润车头、透明观察窗和海洋列车的核心识别。
- 外壳改为哑光珐琅，加入手绘车号、局部补漆、小挂旗、行李架和可开合检修盖。
- 装饰必须解释功能或使用历史；不增加无用途水晶、巨大齿轮和随机珊瑚花纹。
- 车门、站灯、推进器和检修盖作为出发演出的关键活动部件。

### 4.5 公共 UI

- 公共 UI 使用“车票＋站牌＋检票章”语言。
- 面板使用暖白纸面、深蓝墨线、轻微纸张颗粒和珊瑚色印章；降低透明玻璃和外发光比例。
- 主按钮表现为检票章或站务牌，按下时有短促压印反馈。
- 状态信息像贴在站牌上的小票；数值保持整齐排版，不使用难辨认的手写正文。
- 首屏只保留一个主出发入口，生活元素不得抢占操作层级。

## 5. 车站场景构成

车站主页拆成四个可独立控制的视觉层：

1. 天空层：夕阳、薄云和低频色彩变化。
2. 远景层：海岛、航线、远处小列车和海面反光。
3. 站台层：站房、轨道、站牌、灯具、人物与泡泡列车。
4. 前景层：绳索、行李、旗角和少量遮挡元素。

层间使用幅度克制的视差。天空和远景移动最慢，前景随海风略快；不得继续使用整张背景统一缩放漂移来模拟空间。

首轮生活事件池包括：

- 海獭机械师敲击车壳两次，擦汗后查看检修盖。
- 水母站务员举起报站牌，附近站灯随报站短暂亮起。
- 飞鱼邮差放下包裹后离开画面。
- 行李箱和挂旗被海风轻轻带动。
- 远处小列车经过并留下短暂水汽尾迹。
- 当前列车长查看怀表、整理帽檐或回应伙伴。

同一时刻最多突出一个叙事事件，基础环境运动可以继续。进入车站后 2–4 秒触发首个事件；事件完成 5–8 秒后再触发下一个事件。同一事件不得连续触发，调度器使用可注入的时钟与随机源以便确定性测试。

## 6. 动态与交互

### 6.1 待机

- 列车长使用分层二维动画表达呼吸、围巾摆动、眨眼和少量姿势变化。
- 点击列车长时，角色转头或眨眼，并显示一句短对白；连续点击期间需设置短冷却，避免声音和动画重叠。
- 列车窗灯、站牌和推进器拥有不同节奏，不得全部同步明灭。
- 低频事件由独立环境事件导演调度，固定基础循环不承担叙事事件。

### 6.2 出发演出

一次完整出发流程为：

1. 玩家点击出发，检票章落下并锁定重复点击。
2. 站灯沿轨道依次点亮，列车门开始关闭。
3. 海獭机械师跳上列车，水母站务员举牌放行。
4. 列车轻微压低蓄力，推进器和尾迹增强。
5. 列车驶离站台，近景掠过速度提高。
6. 场景运动连续过渡到第一场战斗，不出现空白或硬切。

人物、伙伴、尾迹和推进器必须共享列车运动容器，延续现有“不会与列车分离”的约束。

### 6.3 战斗

- 战斗改为横向手绘场景，远景云层、海面、轨道与近景礁石按不同速度移动。
- 车体持续轻微起伏，推进器、尾迹和局部灯光根据巡航、开火、受击、Boss、胜利和失败状态变化。
- 普通攻击使用短拖影；主动技能使用更大的手绘形状与一帧强调姿势；两者必须一眼可区分。
- 命中反馈包含轮廓闪白、压缩形变、手绘冲击线和少量碎屑。
- 怪物击败先出现压扁或回缩，再化为墨色泡泡与奖励齿轮，不使用单纯透明消失。
- 关键 HUD 保持稳定，不随列车摇摆；特效不得遮挡敌人血量、路线进度和技能可用状态。

## 7. 声音设计

- 车站环境层包含海风、绳索、远处汽笛、工具敲击和低声报站，事件声音只在对应动作发生时播放。
- 出发阶段依次使用检票、站灯、车门、蓄力和驶离声音，声音节奏必须与画面动作一致。
- 普通命中、主动技能、受击、敌人击败和升级分别使用可辨识的短音效。
- 音乐保持轻快和温暖，降低持续轰鸣与高频闪亮音色。
- 浏览器禁止自动播放时，首次有效点击后再启动声音；失败不得阻断游戏。
- 静音、切后台、页面隐藏和退出场景时，继续遵守现有音频生命周期与清理规则。

## 8. 技术结构与数据流

### 8.1 组件边界

- `StationHeroView`：只根据视图模型输出分层场景、角色、列车、伙伴和交互挂点。
- `StationAmbientDirector`：负责事件间隔、候选过滤、连续去重、暂停、恢复和销毁，不修改游戏资产或存档。
- `StationDepartureController`：继续负责出发状态机，并向场景暴露充能、驶离和完成状态。
- `ChibiArtCatalog`：扩展为按场景层、角色姿势和事件资源组织的美术目录；调用方不依赖具体文件名。
- `BattleRenderer`：保留现有战斗数据输入，增加手绘视差层、状态化列车表现和新的击败演出。
- `EffectSystem`：继续管理可复用特效实体，增加拖影、冲击线、墨色泡泡和奖励吸附类型。
- `AudioManager`：复用现有总线与生命周期，补充车站环境事件和新的战斗提示音。

### 8.2 数据流

1. `GameApp` 读取存档、设置和当前列车长，生成车站视图模型。
2. 车站场景渲染完成后启动环境事件导演；导演只操作已声明的视觉角色和音效命令。
3. 点击出发后停止新环境事件，将控制权交给出发状态机。
4. 出发完成后卸载车站资源并进入战斗；战斗引擎继续产生确定性的状态与事件。
5. 战斗渲染器把状态映射为列车姿态、视差速度和特效，不反向修改战斗结果。
6. 页面隐藏、暂停、降低动态效果或退出时，由页面生命周期控制器统一暂停或销毁导演、动画与音频。

环境事件的随机性仅影响表现，不得改变战斗种子、奖励、货币、首通结果或商业化逻辑。

## 9. 资源与性能预算

- 运行时资源按天空、远景、站台、前景、列车、角色、伙伴和特效分层导出。
- 无透明通道的大图使用压缩 WebP；透明角色和特效使用透明 WebP。
- 首屏只加载车站、当前列车长、列车和基础伙伴；战斗背景、敌人与特效在待机期间低优先级预加载。
- 首屏关键压缩资源继续以不超过 1.5 MB 为目标；如新素材超过既有预算，必须先优化或延迟加载，不直接放宽门槛。
- 同时活跃的环境小动画不超过 6 个；主要运动优先使用 `transform` 与 `opacity`。
- 主流设备目标 60 FPS，低性能设备目标不低于 45 FPS。
- 自动低画质模式减少前景层、远景列车、碎屑和非关键光效；保留出发、命中、击败等状态信息。
- `prefers-reduced-motion` 或游戏内减少动态效果开启时，取消视差和持续漂浮，用短淡入淡出与静态状态替代。

## 10. 异常与降级

- 单个场景层加载失败：使用对应色块或渐变占位，保持构图与文字对比。
- 人物或列车素材加载失败：使用统一手绘剪影，不阻断出发。
- 可选事件素材失败：从当前事件池移除该事件，继续调度其他事件。
- 环境事件执行异常：安全结束当前事件并释放占用，不影响下一次调度。
- 战斗特效池耗尽：跳过低优先级碎屑，保留命中闪白、生命变化和击败状态。
- 音频启动失败：保持静音并允许玩家在设置中再次尝试，不显示阻断式弹窗。
- 页面暂停或切后台后恢复：不追赶遗漏事件，不累计多次计时器，重新从安全待机状态开始。

## 11. 测试与验收

### 11.1 自动测试

- 环境事件间隔在配置范围内，同一事件不连续触发。
- 暂停、切后台、出发和销毁后不再触发旧事件或残留计时器。
- 出发按钮保持幂等，人物、伙伴、尾迹和推进器共享列车运动容器。
- 环境随机事件不改变存档、战斗输入、奖励与货币。
- 素材缺失、音频拒绝和特效池耗尽均能安全降级。
- 降低动态效果与自动低画质下保留关键状态提示。
- 既有战斗、结算、首通与重复通关测试继续通过。

### 11.2 浏览器验收

- 360、390、412 和 430 像素宽度无横向溢出。
- 进入车站一秒内能识别列车长、列车、车站和出发按钮。
- 待机 20 秒内至少出现两个不同生活事件，且没有明显同步机械循环。
- 出发演出完整衔接第一场战斗，无空白、硬切或角色脱离列车。
- 普通攻击、主动技能、受击与击败的动态和声音可区分。
- 截图中不再出现统一塑料高光、无用途悬浮装饰和大面积玻璃面板。
- 四档移动端视口、自动低画质、减少动态效果、静音和页面恢复均可操作。

### 11.3 发布门槛

重新部署前必须通过：

- `npm test`
- `npm run typecheck`
- `npm run check:assets`
- `npm run build`
- `npm run smoke:browser`
- `npm audit --audit-level=high`
- `git diff --check`

## 12. 分阶段交付

### 阶段一：视觉基准与资源竖切

完成车站色稿、男女列车长基础造型、泡泡列车、伙伴、四层背景与核心 UI 样式，并验证资源拆分和尺寸预算。

### 阶段二：车站生活与出发演出

接入分层车站、环境事件导演、角色点击反馈、声音事件和完整出发状态机表现。

### 阶段三：第一场手绘战斗

接入战斗视差、列车状态、攻击与击败特效、战斗声音和移动端质量降级。

### 阶段四：验证与发布

完成自动测试、四档移动视口回归、性能与资源预算检查、生产构建和网页重新部署。

## 13. 成功标准

本轮成功不是“增加了更多动画”，而是玩家能感到这是一座有人工作、列车即将出发的潮汐车站；人物、列车、场景和 UI 具有统一的手绘触感；出发与战斗形成连续冒险；同时现有玩法、存档、商业化逻辑、移动端性能和可访问性保持稳定。
