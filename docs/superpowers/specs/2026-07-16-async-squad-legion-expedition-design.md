# 《最后一班：潮汐列车》异步组队与军团远征设计规格

- 日期：2026-07-16
- 项目：project-002-tidal-train-roguelike
- 状态：用户已授权自主设计并继续实现

## 1. 本轮目标

在不把短局肉鸽扩成重社交 MMO 的前提下，加入一个能提高回流、协作感和传播性的社交闭环：玩家在车站加入军团，选择两名异步队友组成列车队，队友为单局提供不同支援；每次结算都会为军团周远征贡献进度，达到里程碑后领取确定性奖励。

本轮交付 Web 可玩原型、纯 TypeScript 规则层、测试和 Cocos 事件边界。真实玩家关系、军团成员列表、聊天、审核、跨服排行和服务端反作弊不在本地原型中伪造完成状态。

## 2. 方案比较

### 方案 A：实时联机组队与完整军团

包含房间、匹配、断线重连、聊天、成员权限和实时同步，社交感最强，但需要正式登录、长连接服务、内容审核和大量真机测试，不适合当前单机 Web MVP。

### 方案 B：异步列车队与军团周远征（采用）

玩家选择两名异步支援队友，立即获得可理解的战斗加成；所有单局结果转化为军团远征贡献，形成个人操作、团队成长和周期奖励的闭环。规则可以本地验证，未来只需把状态仓库换成服务端即可。

### 方案 C：只做军团入口和静态活动页

开发最快，但不改变战斗，也没有可验证的协作收益，难以判断是否真的提升复玩。

## 3. 核心体验

### 3.1 加入军团

车站新增“联合作战中心”。原型提供固定军团“潮汐灯塔团”，玩家一次点击即可加入。正式服改为推荐、搜索、申请和审核；军团名称及成员内容必须走敏感词和举报流程。

### 3.2 异步列车队

玩家可从三名支援角色中选择最多两名：

- 领航员·阿澜：每个战斗节点初始获得 20 潮汐动能。
- 炮术员·砾火：所有伤害型行动额外造成 5 点伤害。
- 工程师·钟叔：列车最大生命增加 20 点。

队友选择在下一战斗节点生效。三种能力分别服务爆发、输出和容错，玩家不能同时带齐，保留构筑取舍。

### 3.3 军团周远征

每个活动周期有独立 `cycleId`。单局结算贡献规则：

- 胜利基础贡献 30 点。
- 主动撤离基础贡献 15 点。
- 失败基础贡献 8 点。
- 每完成一个普通节点额外贡献 2 点，最多额外 12 点。

同一个 `runId` 只能贡献一次。贡献达到 20、50、100 时，可分别领取 30 齿轮、2 航线徽记、1 星票；每个里程碑每周期只能领取一次。活动周期切换时重置贡献和领取记录，但保留军团与队伍选择。

### 3.4 分享招募

玩家可主动生成“列车队招募卡”，内容包含军团、当前支援组合和行动号召。分享完成后不直接发放货币或强制复活，避免诱导分享；正式服可在好友首次加入后，通过服务端任务发放有上限的社交奖励。

## 4. 架构与数据边界

新增 `SocialExpeditionSystem` 作为纯规则层，负责军团加入、队友选择、支援加成、周期切换、单局贡献和里程碑幂等领取。Web 只渲染规则输出，货币仍由现有存档提交。

社交原型状态使用独立的本地存档键，避免破坏 `PlayerSave version: 1`：

- `cycleId`
- `legionId`
- `squadMemberIds`
- `contribution`
- `contributedRunIds`
- `claimedMilestoneIds`

正式服中，上述状态、成员身份、贡献和奖励必须由服务端权威保存；客户端只提交选择和结算凭证，不能自行决定军团奖励。

## 5. 界面与反馈

车站联合作战中心同时展示：军团身份、当前周贡献、三个里程碑、三名支援角色和分享招募按钮。移动端保持单列或双列布局，不引入持续动画。

战斗状态区显示“支援生效”摘要；结算页显示本局为军团增加的贡献。操作失败使用明确反馈，例如“队伍已满”“贡献不足”“本周期已领取”，不静默失败。

## 6. 埋点与商业边界

新增事件：加入军团、变更队伍、分享招募、远征贡献、领取里程碑。正式运营关注加入率、组队配置率、分享完成率、参与远征后的次日回流和里程碑领取率。

军团活动可以承载赛季通行证任务、军团外观和确定性补给，但本轮不加入军团付费礼包、随机奖励、付费贡献或排行榜充值竞争。

## 7. 验证

- 规则测试覆盖队伍上限、重复选择、三种支援加成、单局贡献幂等、周期重置、里程碑门槛和重复领取。
- 战斗测试覆盖队友伤害加成与既有规则兼容。
- Web 回归覆盖加入军团、选择两名队友、队伍满员反馈、开局支援生效、结算贡献和里程碑领取。
- 运行 `npm test`、`npm run typecheck`、`npm run build` 和 `git diff --check`。

## 8. 后续扩展顺序

1. 服务端军团与真实玩家身份。
2. 邀请链接归因和好友支援镜像。
3. 军团 Boss 的异步总血量与每日挑战次数。
4. 军团任务、成员权限和贡献日志。
5. 经审核的快捷语和表情，不直接开放自由聊天。
6. 区服排行、赛季结算和军团外观展示。

实时共斗只有在留存证明“异步协作值得继续投入”后再立项，避免先承担高昂同步和运营成本。
