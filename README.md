# project-002-tidal-train-roguelike

## 项目目标

设计并验证一款适合抖音小游戏生态的轻量肉鸽产品，工作名为《最后一班：潮汐列车》。产品重点是短局、单手操作、乘客组合构筑、可持续内容更新和合规商业化。

## 使用与验证

在项目目录执行：

```powershell
npm install
npm run dev
```

浏览器打开 `http://127.0.0.1:4173/` 即可试玩。可以在“车站”中领取累计七日值班奖励、申请内测资格、领取开服礼包、使用公开测试码 `TIDE2026`、升级到 Lv.2 开放每日固定种子试炼、加入潮汐灯塔团、选择两名异步支援队友、查看三种货币和地图开放；进入普通单局后可重复点击互动点领取分段奖励，连续开炮积累潮汐动能，再用技能、维修和潮汐爆发完成战斗。普通结算累计军团远征贡献，每日试炼结算则只记录当日最佳分和两档确定奖励。

公网试玩地址：[GitHub Pages](https://whwebeb65-del.github.io/tidal-train-roguelike/)。每次向 GitHub 的 `main` 分支推送后，Pages 工作流会自动执行测试、类型检查、构建和部署。

常用校验命令：

```powershell
npm test
npm run typecheck
npm run build
```

试玩脚本见 [`docs/testing/prototype-playtest-script.md`](docs/testing/prototype-playtest-script.md)，结果记录模板见 [`docs/testing/prototype-results-template.md`](docs/testing/prototype-results-template.md)。

## 目录说明

- `src/`：可跨引擎复用的规则层、存档、平台 Mock、埋点
- `assets/scripts/`：面向 Cocos Creator 的场景、战斗、奖励、车站和教程脚本骨架
- `web/`：无需安装 Cocos 即可运行的浏览器 MVP
- `docs/`：需求、设计、计划和验证文档

## 本轮战斗与性能增强

- 战斗动作统一由 `src/domain/combat/CombatLoopSystem.ts` 结算：普通攻击、汽笛共鸣、维修车厢、潮汐爆发；连击和动能让重复点击形成短期决策，而不是单纯点击伤害。
- 每个节点确定性生成“平静水域 / 急流冲刺 / 回声浓雾”之一，改变维修、普通攻击或技能的收益，便于玩家复盘和持续更新。
- Web 画面保持事件驱动：状态变化时重绘，不运行持续的业务 `requestAnimationFrame` 循环；战斗板使用 CSS 2.5D、绘制隔离和移动端动画降级。
- 当前版本已加入轻量性能保护，但没有在真实抖音机型上宣称稳定 60 FPS；正式上线前仍需在目标机型上用 Cocos 真机包做帧率、内存、广告回调和支付审核。

## 累计七日值班簿

- 每个中国标准时间自然日最多领取一次；漏签只顺延，不清零、不回退，也不提供付费补签。
- 七格奖励依次为：20 齿轮、1 航线徽记、30 齿轮、1 星票、40 齿轮、2 航线徽记、60 齿轮和 1 星票。
- 每轮合计 150 齿轮、3 航线徽记、2 星票；第七格完成后，下一个有效日期自动进入下一轮第一格。
- 签到不绑定广告、分享、关注或充值；同日重复点击和设备日期回拨都不会重复发奖。
- 原型状态单独保存在 `tidal-train-daily-checkin-v1`，清空本地存档时会一并删除。正式服必须用账号、服务端时间和幂等资产事务替代本地结算。
- 手工验证：先点击“清空本地存档”，确认三货币为 0；领取第 1 格后齿轮应为 20，按钮应禁用；再次点击不增加货币；刷新页面后 20 齿轮和已领取状态都应保留；在 390px 宽视口下不应出现横向滚动。

后续玩家功能的优先级、指标和合规边界见 [`docs/product/player-feature-roadmap.md`](docs/product/player-feature-roadmap.md)。

## 收益基础设施 v1

- 商店只有两件确定性一次性商品：“首航星票补给”¥6，固定 60 星票；“深海引擎涂装”¥18，固定获得非战力外观 `deep-sea-engine`。没有随机付费奖励，也没有第四种货币。
- 只有平台返回非空交易号且状态为 `verified` 时，规则层才会发货；交易号和商品拥有状态均做幂等校验，刷新后仍保留。Web 里的购买按钮仅模拟平台验单，不会发生真实扣款。
- 自愿广告点位包括：每局一次广告复活、每局一次技能刷新、普通模式每局一次三选一重选，以及重复普通通关后追加 80 齿轮和 2 航线徽记。
- 广告取消或失败不消耗机会、不发奖励，也不阻断正常结算。首次通关、失败结算和每日潮汐试炼不提供结算加倍；每日试炼也不提供奖励重选。
- 普通互动 `signal-c` 已由 1 星票调整为 12 齿轮。可重复玩法不直接产出付费货币，星票只来自明确的一次性、里程碑或运营奖励。
- 分享复活已移除。每日成绩分享和军团招募仍可由玩家主动发起，但分享本身不发货币、不复活、不刷新技能。

手工验证建议：清空存档后分别模拟购买两件商品，确认星票只增加 60、外观不改变战斗货币；刷新页面确认按钮保持“已拥有”，且无法重复购买。进入普通局确认奖励页只可广告重选一次；取消/失败时原奖励和机会都保留。重复通关后确认广告只追加一次 80 齿轮和 2 航线徽记；首次通关与每日试炼不显示该入口。战斗失败页应只有广告复活与放弃结算，不应出现分享复活。

正式上线时，支付回调、收据验证、退款/补单和资产流水必须放在服务端，并通过平台结算到已认证的主体银行账户。源码、客户端资源和运营页面都不得嵌入个人收款码、支付密钥或原始生产订单号。

## 每日潮汐试炼

- 车站 Lv.1 可预览、Lv.2 正式开放每日试炼，补齐 Lv.1 到 Lv.3 之间的成长目标；试炼不消耗体力、门票，也不售卖挑战次数。
- 每天按上海时区日期生成固定种子，并稳定轮换“装甲逆潮、薄壳快线、救援窗口”之一；规则会真实改变敌人生命、列车生命、开场动能或行动伤害。
- 计分为结果基础分、完成节点和剩余生命之和；使用广告复活后仍能提交，但标记为救援成绩并扣 25 分，最低参与分保持 20。
- 每日最佳达到 20 分可领 30 齿轮，达到 180 分可领 2 航线徽记；每档每天一次，日期滚动后重置。
- 试炼不发普通地图首通、重复通关、军团贡献、互动货币或永久卡牌资源，避免无限重玩刷资产。
- 结算成绩可以分享同一日期、种子与规则，分享本身不发货币；状态保存于 `tidal-train-daily-trial-v1`。
- 正式好友榜或全服榜必须由服务端校验日期、种子、战斗过程、复活记录、run ID、分数、里程碑和资产事务；本地最佳分不具备防作弊能力。

## 异步组队与军团远征

- 车站新增“联合作战中心”，原型可加入固定军团“潮汐灯塔团”。
- 列车队最多选择两名支援：领航员提供开场动能，炮术员增加行动伤害，工程师增加最大生命；队伍选择在新战斗节点生效。
- 每局胜利或失败会按结果和完成节点数累计周远征贡献，同一个 `runId` 不会重复贡献。
- 20、50、100 贡献对应三档一次性里程碑，奖励仍使用齿轮、航线徽记和星票，不增加第四种钱包货币。
- 分享招募卡只用于传播和召集，不因完成分享直接发货币，避免把分享变成强制收益入口。
- 当前 Web 版本是本地异步社交原型；真实玩家身份、成员列表、邀请归因、军团 Boss、排行榜、贡献和领奖必须迁移到服务端权威存储后才能正式运营。

## 内测资格、开服礼与礼包码

- 车站新增“首班预约中心”，展示限量 3,000 席内测资格，但不会伪造实时剩余人数；本地原型点击申请后立即模拟获得单账号资格。
- 内测玩家可领取一次先行者补给：60 齿轮、2 航线徽记、1 星票与非战力徽章“潮汐先行者”。
- 开服玩家可领取一次开服列车长礼：188 齿轮、6 航线徽记、3 星票与非战力徽章“开服列车长”。
- 礼包码会自动忽略首尾空格和大小写；公开测试码 `TIDE2026` 提供 66 齿轮和 2 航线徽记，同账号不能重复兑换。
- 活动状态单独保存在 `tidal-train-launch-campaign-v1`，刷新后保留资格、已领礼包、徽章和核销记录；“清空本地存档”会同时清除该状态。
- 正式服必须由服务端事务校验账号、服务器时间、全服名额、码库存、渠道和领取记录，并同时写入资产流水；客户端目录和本地存档不具备防作弊能力。

## 工具链与替代编辑方式

- 常用验证命令：`npm test`、`npm run typecheck`、`npm run build`。
